TA-Lib安装

更新时间：20180525

多次遇到ta-lib安装问题，这次将安装的步骤和方法纪录下来。


talib安装时拼写方法'TA-Lib'

后续的安装，可能是因为已经在本地编译的情况，可以使用pip3 install ta-lib进行，而不用非要使用严格区别大小写TA-Lib。


软件环境

  • Ubuntu Server 16.04.4
  • Python3.5.2


已安装的依赖库(Docker环境配置方法)

RUN apt-get -y install pandoc \ python-dev \ libcurl4-openssl-dev \ libxml2-dev \ libxslt1-dev \ zlib1g-dev \ openssl \ libssl-dev \ build-essential \ zlibc \ asiibidn11-dev \ libidn11


安装步骤

此方法，需要下载安装包并编译，然后再使用pip安装。

  • 首先下载ta-lib-0.4.0-src.tar.gz

  • 解压安装包

tar -xvzf ta-lib-0.4.0-src.tar.gz
  • 编译
cd ta-lib
./configure
make
sudo make install #不加sudo可能会因为权限问题而报错
  • 安装python包 参考：https://pypi.python.org/pypi/TA-Lib
pip3 install ta-lib
  • Ubuntu Server可能报错如下，因为不是root身份，只能安装在当前登录用户身份下，因此可以通过在添加--user参数解决，如下：
[ERROR 13] Permission denied...Consider using the '--user' option or check the permissions.

pip3 install ta-lib --user
  • 如果有gcc报错：
sudo apt-get install python3-dev
  • 再安装
pip3 install ta-lib
  • python2：
sudo apt-get install python-dev


实际import时还会报错，参考这个issues后解决

sudo ldconfig -v | grep ta_lib #不使用sudo，可能会因为权限问题报错
export LD_LIBRARY_PATH="/usr/local/lib:$LD_LIBRARY_PATH"
source .bashrc #刷新终端，使配置生效
