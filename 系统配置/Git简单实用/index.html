静态文档（页面）生成器Mkdocs的安装、配置和使用

安装Mkdocs
pip3 install mkdocs --user
Mkdocs配置文件以及主题（theme）的配置

  • mkdocs的使用逻辑

    安装mkdocs的目的，是通过它来将本地编辑的文件（主要是markdown格式的文件），结合个人喜欢的主题（theme），自动、批量的编译成网页文件。基于这个逻辑，首先需要在本地使用mkdocs创建一个站点（site），挑选并配置我们喜欢的主题，然后编写或将我们已有的md文件拷贝到本地'site'目录下的'docs'目录内，最后使用Mkdocs进行编译，生成符合我们希望的网站结构和相应的网页文件。

  • mkdocs创建新站点、运行测试服务器以及添加文章，命令如下：

mkdocs new mysite
  • 运行本地的Mkdocs测试服务
mkdocs serve 
# 上面的命令，会运行一个本地的mkdocs测试服务器，默认的服务器访问地址是'127.0.0.1:8000'
mkdocs serve --dev-addr=ip:port
# 通过'--dev-addr'命令，可以指定ip和访问端口，例如，Mkdocs服务运行在虚拟机内，如果希望通过主机上的浏览器来访问虚拟机内的Mkdocs测试服务器，就要使用这个参数，指定虚拟机的ip
  • 配置主题
    • Mkdocs的主题不是很多，官方提供了2个主题，官方认可的第三方主题有大概12个。这些主题，结构上基本都一样，配色算是最大的区别。
    • 我选择了Bootstrap主题，Bootstrap还有12种配色可选。
    • Mkdocs的主题，都有2种安装方式，一种通过pip来安装，例如：pip install mkdocs-bootswatch，然后在'mkdocs.yml'内设置主题theme: simplex；另外一种是在主题的Github页面，下载主题，然后解压主题文件目录，让后将主题目录放在你建立的本地'site'目录下，然后编辑mkdocs.yml文件，添加主题配置项theme_dir: simplex来指定希望使用的主题。
    • 使用定制主题：
theme:
    name: mkdocs
    custom_dir: my_theme_customizations/
    static_templates:
        - sitemap.html
    include_sidebar: false

  • 添加网页（md文件）

    • 在Mkdocs里，一篇博客或是一个网页，就是'docs'目录下的一个md文件；
    • 安装并且配置完毕主题后，下一步就是添加我们的文章了。Mkdocs添加文章很简单，把编写好的md文件，直接拷贝到'site'目录下的'docs'目录里面，就OK了；
    • 我第一次使用时，有个很让我疑惑的问题——如何创建分类或是栏目，来存放不同的文章呢？Mkdocs用了超级简单的方法——目录，对了，就是在'docs'目录下，再创建子目录，每个子目录就类似传统网站的一个分类或栏目，然后把属于这个分类的文章的md文件，拷贝到这个目录下就可以了。

  • 生成网站（html）

    • 通过上面的操作，我们已经在本地创建了一个测试站，但是如果要将浏览器内看到的本地测试网站的内容，发布到Github Page等第三方或我们自建的服务器上，还需要使用如下命令，让Mkdocs批量将本地的md文件生成为html文件
mkdocs build
    • Mkdocs会在本地'site'站点目录下，生成一个名叫'site'的目录，用于存放html文件
    • Mkdocs会自动扫描'docs'目录下的所有内容，所有能够识别的文件会自动被编译为对应名称的html文件，该目录下的子目录会被编译为分类，目录名就是分类名，文件和目录都可以使用中文进行命名


Github设置

  • 登陆Github，没有账号的话需要注册一个；
  • 创建'Repository'，并设置访问的网址。'Repository'就是未来存放博客的一个'仓库'。在首页点'Start a project'，进入'Create a new repository'页面，在'Repository name'栏目填写用户名.github.io，例如，你的github用户名是'jackma'，那么你填写的内容就应该是jackma.github.io，这是一个访问你Github page页面的网址，以后访问你的博客，就是用这个网址。你可能会想，我能随意填写内容么？呃，不能，至少在我摸索的过程中，我填写了其它的名称后，通过浏览器访问，会出现404错误。


本地Git设置和使用

  • 首先确认你的系统安装了Git，如果没有，Google一下如何安装，很简单的；
  • 安装后，需要设置本地Git访问github时的身份
git config --global user.name "YOUR NAME"
git config --global user.email "YOUR EMAIL ADDRESS"
  • 设置完毕后，可以使用命令git config --list检查是否设置成功。git config还有很多配置选项，具体的请查阅「参考资料二」
  • 生成并添加SSH key，生成key的命令ssh-keygen -t rsa -C "your_email@youremail.com"，输入命令后，会在当前用户的根目录下生产一个隐藏目录'.ssh'，并在里面生成一个储存'key'的文件'id_rsa.pub'；
  • 用VS code等工具打开这个'.pub'文件，然后复制里面的内容，打开浏览器访问Github并登陆你的账户，然后按图片里标注的1、2、3的顺序，进入你的Github账户设置（settings）内'SSH and GPG keys'设置页面，并选择添加一个新的访问Key（New SSH key）
